## О программе encfsy
encfsy — это реализация EncFS для Windows на базе **Dokany** и **Crypto++**.
Работает **только на 64-битных системах**.

Программа шифрует имена файлов и их содержимое, сохраняя структуру каталогов.
Это идеально подходит для синхронизации зашифрованных данных с Dropbox, Google Drive, rsync или другими удалёнными хранилищами: файлы остаются зашифрованными сквозным шифрованием, и администраторы хранилища не могут видеть их содержимое.

## ⚠️ При двусторонней облачной синхронизации не используйте `--paranoia`
В режиме `--paranoia` (внешняя цепочка IV) любые переименования, сделанные утилитой синхронизации, приводят к тому, что даже после возврата имён **содержимое остаётся нечитаемым мусором**.
- Для двусторонней синхронизации (Dropbox/OneDrive/Google Drive и т.д.) **не включайте `--paranoia`**.
- Если режим нужен, ограничьте его односторонним бэкапом, где имена не меняются.

## Обработка конфликтных файлов облака

При использовании облачных хранилищ (Dropbox, Google Drive, OneDrive) конфликты синхронизации могут создавать файлы со специальными суффиксами, которые невозможно расшифровать обычным способом. Опция `--cloud-conflict` включает обнаружение и обработку таких конфликтных файлов.

**Поддерживаемые шаблоны конфликтов:**
- Dropbox: `имя_файла (конфликтная копия компьютера 2024-01-01).ext`
- Google Drive: `имя_файла_conf(1).ext`

**Использование:**
```bash
encfs.exe C:\Data M: --cloud-conflict
```

**Примечание:** Эта опция отключена по умолчанию, поскольку обнаружение конфликтов может незначительно влиять на производительность и требуется только при использовании облачных сервисов синхронизации.

## Сканирование недействительных имён файлов

Опция `--scan-invalid` сканирует зашифрованный каталог и сообщает об именах файлов, которые невозможно расшифровать. Результаты выводятся в формате JSON.

**Использование:**
```bash
encfs.exe C:\encrypted --scan-invalid
encfs.exe C:\encrypted --scan-invalid --cloud-conflict  # Сканировать с обнаружением облачных конфликтов
```

**Формат вывода JSON:**
```json
{
  "invalidFiles": [
    {
      "fileName": "зашифрованное_имя_файла",
      "encodedParentPath": "encDir1\\encDir2",
      "decodedParentPath": "dir1\\dir2"
    }
  ],
  "totalCount": 1
}
```

## Функции безопасности
encfsy использует **Диспетчер учётных данных Windows** для безопасного управления паролями.

- Пароли шифруются с помощью **DPAPI** (Data Protection API) и привязываются к текущей учётной записи пользователя
- Устраняет необходимость передачи паролей через stdin между GUI и encfs.exe, исключая риски перехвата
- Опция «Запомнить пароль» сохраняет пароли для автоматического ввода при следующем запуске
- Пароли **хранятся отдельно для каждого зашифрованного каталога (rootDir)**

### Где хранятся пароли
Сохранённые пароли можно просмотреть в Панели управления → Диспетчер учётных данных → Учётные данные Windows.
Они отображаются с именами вида `EncFSy:c:\path\to\encrypted`.

## Использование GUI
Используйте **encfsw.exe** для удобного монтирования и размонтирования томов без командной строки.

1. Выберите зашифрованный каталог (rootDir)
2. Выберите букву диска для монтирования
3. Введите пароль (отметьте «Remember Password» для сохранения)
4. Нажмите «Mount»

«Show Advanced Options» предоставляет доступ к тем же детальным настройкам, что и версия командной строки.
Расширенная опция «NameIO Stream» выбирает шифрование имён nameio/stream только при создании нового тома. Существующие тома продолжают использовать настройку из `.encfs6.xml`.

## Использование Диспетчера учётных данных из командной строки
Когда вы монтируете с отмеченной опцией «Remember Password» в GUI, пароль сохраняется в Диспетчере учётных данных Windows.
Затем вы можете монтировать из командной строки без ввода пароля, используя опцию `--use-credential`.

```bash
# 1. Сначала смонтируйте через GUI с отмеченной опцией «Remember Password»
#    → Пароль сохраняется в Диспетчере учётных данных

# 2. Затем можно монтировать из командной строки без запроса пароля
encfs.exe C:\Data M: --use-credential
```

## Ограничение длины имени файла
encfsy использует современный API *длинных путей*, поэтому традиционное ограничение **MAX_PATH** в 260 символов для полных путей **не применяется**.

Однако NTFS по-прежнему ограничивает каждый компонент пути (имя папки или файла) **255 символами UTF-16**.
Поскольку шифрование увеличивает имена примерно на 30%, держите **каждое имя файла не более 175 символов**, чтобы оставаться в пределах ограничения компонента и сохранять совместимость с инструментами, не поддерживающими длинные пути.

## Использование

```
Использование: encfs.exe [опции] <rootDir> <mountPoint>

Аргументы:
  rootDir      (напр., C:\test)              Каталог для шифрования и монтирования
  mountPoint   (напр., M: или C:\mount\dokan) Точка монтирования - буква диска (напр., M:\)
                                              или пустая папка NTFS

Опции:
  -u <mountPoint>                              Размонтировать указанный том
  -l                                           Показать смонтированные тома Dokan
  -v                                           Отправить отладочный вывод в отладчик
  -s                                           Отправить отладочный вывод в stderr
  -i <мс>              (по умолч.: 120000)     Таймаут (мс) до отмены операции и размонтирования
  --use-credential                             Читать пароль из Диспетчера учётных данных Windows
                                               (пароль сохраняется)
                                               Примечание: Сначала нужно сохранить пароль через GUI
                                               с отмеченной опцией «Remember Password»
  --use-credential-once                        Читать пароль из Диспетчера учётных данных Windows
                                               (удалить после чтения, одноразовое использование)
  --scan-invalid                               Сканировать зашифрованный каталог и сообщать об именах
                                               файлов, которые невозможно расшифровать. Вывод в JSON
  --dokan-debug                                Включить отладочный вывод Dokan
  --dokan-network <UNC>                        UNC-путь для сетевого тома (напр., \\host\myfs)
  --dokan-removable                            Представить том как съёмный носитель
  --dokan-write-protect                        Монтировать файловую систему только для чтения
  --dokan-mount-manager                        Зарегистрировать том в Диспетчере монтирования Windows
                                               (включает поддержку Корзины и т.д.)
  --dokan-current-session                      Показывать том только в текущем сеансе
  --dokan-filelock-user-mode                   Обрабатывать LockFile/UnlockFile в пользовательском режиме;
                                               иначе Dokan управляет ими автоматически
  --dokan-enable-unmount-network-drive         Разрешить размонтирование сетевого диска через Проводник
  --dokan-dispatch-driver-logs                 Пересылать логи драйвера ядра в userland (медленно)
  --dokan-allow-ipc-batching                   Включить пакетную обработку IPC для медленных ФС
                                               (напр., удалённое хранилище)
  --public                                     Имперсонировать вызывающего пользователя при открытии
                                               дескрипторов в CreateFile. Требуются права администратора
  --allocation-unit-size <байт>                Размер единицы распределения тома
  --sector-size <байт>                         Размер сектора тома
  --volume-name <имя>                          Имя тома в Проводнике (по умолч.: EncFS)
  --volume-serial <hex>                        Серийный номер тома в hex (по умолч.: из базового)
  --paranoia                                   Включить AES-256, переименование IV и внешнюю цепочку IV
  --alt-stream                                 Включить альтернативные потоки данных NTFS
  --case-insensitive                           Сопоставление имён файлов без учёта регистра
  --nameio-stream                              При создании нового тома использовать nameio/stream
                                               для шифрования имён (по умолч.: nameio/block)
  --cloud-conflict                             Включить обработку конфликтных файлов облака
                                               (Dropbox, Google Drive, OneDrive). По умолч. выкл.
  --reverse                                    Обратный режим: показать plaintext rootDir
                                               зашифрованным в mountPoint

Примеры:
  encfs.exe C:\Users M:                                    # Смонтировать C:\Users как диск M:\
  encfs.exe C:\Users C:\mount\dokan                        # Смонтировать в папку NTFS C:\mount\dokan
  encfs.exe C:\Users M: --dokan-network \\myfs\share       # Как сетевой диск с UNC \\myfs\share
  encfs.exe C:\Data M: --volume-name "Безопасный диск"     # С пользовательским именем тома
  encfs.exe C:\Data M: --use-credential                    # Использовать сохранённый пароль из Диспетчера
  encfs.exe C:\Data M: --nameio-stream                     # Создать новый том с nameio/stream
  encfs.exe C:\Data M: --cloud-conflict                    # Монтировать с поддержкой облачных конфликтов
  encfs.exe C:\encrypted --scan-invalid                    # Сканировать недействительные имена (JSON)

Для размонтирования нажмите Ctrl+C в этой консоли или выполните:
  encfs.exe -u <mountPoint>
```
