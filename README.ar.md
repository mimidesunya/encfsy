## حول encfsy
encfsy هو تطبيق EncFS لنظام Windows يعمل بواسطة **Dokany** و **Crypto++**.
يعمل **حصرياً على أنظمة 64 بت**.

يقوم البرنامج بتشفير أسماء الملفات ومحتوياتها مع الحفاظ على بنية شجرة الدليل.
هذا يجعله مثالياً لمزامنة البيانات المشفرة مع Dropbox أو Google Drive أو rsync أو التخزين البعيد الآخر: تبقى الملفات مشفرة من طرف إلى طرف، لذا لا يمكن لمسؤولي التخزين رؤية محتوياتها.

## ⚠️ لا تستخدم وضع `--paranoia` مع مزامنة سحابية ثنائية الاتجاه
عند تفعيل وضع `--paranoia` (سلسلة IV الخارجية)، أي تغيير في أسماء الملفات بواسطة أداة المزامنة سيجعل المحتوى **غير قابل لفك التشفير** حتى لو أعدت الأسماء يدوياً.
- في سيناريوهات المزامنة ثنائية الاتجاه (Dropbox/OneDrive/Google Drive وغيرها)، **لا تفعّل `--paranoia`**.
- إذا اضطررت لاستخدامه، فاقصره على النسخ الاحتياطي أحادي الاتجاه حيث لا تتغير أسماء الملفات.

## معالجة ملفات تعارض السحابة

عند استخدام خدمات التخزين السحابي (Dropbox، Google Drive، OneDrive)، قد تنشئ تعارضات المزامنة ملفات بلاحقات خاصة لا يمكن فك تشفيرها بالطريقة العادية. خيار `--cloud-conflict` يمكّن اكتشاف ومعالجة ملفات التعارض هذه.

**أنماط التعارض المدعومة:**
- Dropbox: `اسم_الملف (نسخة متعارضة من الكمبيوتر 2024-01-01).ext`
- Google Drive: `اسم_الملف_conf(1).ext`

**الاستخدام:**
```bash
encfs.exe C:\Data M: --cloud-conflict
```

**ملاحظة:** هذا الخيار معطل افتراضياً لأن اكتشاف التعارض قد يكون له تأثير طفيف على الأداء ولا يُحتاج إليه إلا عند استخدام خدمات المزامنة السحابية.

## فحص أسماء الملفات غير الصالحة

خيار `--scan-invalid` يفحص الدليل المشفر ويبلغ عن أسماء الملفات التي لا يمكن فك تشفيرها. يتم إخراج النتائج بتنسيق JSON.

**الاستخدام:**
```bash
encfs.exe C:\encrypted --scan-invalid
encfs.exe C:\encrypted --scan-invalid --cloud-conflict  # فحص مع اكتشاف تعارض السحابة
```

**تنسيق إخراج JSON:**
```json
{
  "invalidFiles": [
    {
      "fileName": "اسم_الملف_المشفر",
      "encodedParentPath": "encDir1\\encDir2",
      "decodedParentPath": "dir1\\dir2"
    }
  ],
  "totalCount": 1
}
```

## ميزات الأمان
يستخدم encfsy **مدير بيانات اعتماد Windows** لإدارة كلمات المرور بشكل آمن.

- يتم تشفير كلمات المرور باستخدام **DPAPI** (واجهة برمجة تطبيقات حماية البيانات) وربطها بحساب المستخدم الحالي
- يلغي الحاجة إلى تمرير كلمات المرور عبر stdin بين واجهة المستخدم الرسومية و encfs.exe، مما يزيل مخاطر الاعتراض
- خيار "تذكر كلمة المرور" يحفظ كلمات المرور للإدخال التلقائي عند التشغيل التالي
- يتم تخزين كلمات المرور **بشكل منفصل لكل دليل مشفر (rootDir)**

### أين يتم تخزين كلمات المرور
يمكن عرض كلمات المرور المحفوظة في لوحة التحكم ← مدير بيانات الاعتماد ← بيانات اعتماد Windows.
تظهر بأسماء مثل `EncFSy:c:\path\to\encrypted`.

## استخدام واجهة المستخدم الرسومية
استخدم **encfsw.exe** لتركيب وإلغاء تركيب وحدات التخزين بسهولة دون سطر الأوامر.

1. حدد الدليل المشفر (rootDir)
2. اختر حرف محرك الأقراص للتركيب
3. أدخل كلمة المرور (حدد "Remember Password" للحفظ)
4. انقر على "Mount"

"Show Advanced Options" توفر الوصول إلى نفس الإعدادات التفصيلية الموجودة في إصدار سطر الأوامر.
الخيار المتقدم "NameIO Stream" يحدد تشفير أسماء الملفات nameio/stream عند إنشاء وحدة تخزين جديدة فقط. تحتفظ وحدات التخزين الموجودة بالإعداد المحفوظ في `.encfs6.xml`.

## استخدام مدير بيانات الاعتماد من سطر الأوامر
عندما تقوم بالتركيب مع تحديد "Remember Password" في واجهة المستخدم الرسومية، يتم حفظ كلمة المرور في مدير بيانات اعتماد Windows.
يمكنك بعد ذلك التركيب من سطر الأوامر دون إدخال كلمة مرور باستخدام خيار `--use-credential`.

```bash
# 1. أولاً، قم بالتركيب عبر واجهة المستخدم الرسومية مع تحديد "Remember Password"
#    ← يتم حفظ كلمة المرور في مدير بيانات الاعتماد

# 2. بعد ذلك، يمكن التركيب من سطر الأوامر دون طلب كلمة المرور
encfs.exe C:\Data M: --use-credential
```

## حد طول اسم الملف
يستخدم encfsy واجهة برمجة تطبيقات *المسار الطويل* الحديثة، لذا فإن حد **MAX_PATH** التقليدي البالغ 260 حرفاً للمسارات الكاملة **لا ينطبق**.

ومع ذلك، لا يزال NTFS يحد كل مكون مسار (اسم مجلد أو ملف) بـ **255 حرف UTF-16**.
نظراً لأن التشفير يزيد الأسماء بحوالي 30%، احتفظ **بكل اسم ملف أقل من 175 حرفاً** للبقاء ضمن حد المكون والحفاظ على التوافق مع الأدوات التي لا تدعم المسار الطويل.

## الاستخدام

```
الاستخدام: encfs.exe [خيارات] <rootDir> <mountPoint>

المعاملات:
  rootDir      (مثال: C:\test)              المجلد المراد تشفيره وتركيبه
  mountPoint   (مثال: M: أو C:\mount\dokan)  موقع التركيب - حرف محرك أقراص (مثل M:\)
                                             أو مجلد NTFS فارغ

الخيارات:
  -u <mountPoint>                              إلغاء تركيب وحدة التخزين المحددة
  -l                                           عرض وحدات تخزين Dokan المركبة حالياً
  -v                                           إرسال مخرجات التصحيح إلى المصحح
  -s                                           إرسال مخرجات التصحيح إلى stderr
  -i <مللي ثانية>       (افتراضي: 120000)      مهلة (بالمللي ثانية) قبل إلغاء العملية وإلغاء التركيب
  --use-credential                             قراءة كلمة المرور من مدير بيانات الاعتماد
                                               (يتم الاحتفاظ بكلمة المرور)
                                               ملاحظة: يجب حفظ كلمة المرور أولاً عبر واجهة المستخدم
                                               مع تحديد "Remember Password"
  --use-credential-once                        قراءة كلمة المرور من مدير بيانات الاعتماد
                                               (حذفها بعد القراءة، استخدام لمرة واحدة)
  --scan-invalid                               فحص الدليل المشفر والإبلاغ عن أسماء الملفات
                                               التي لا يمكن فك تشفيرها. الإخراج بتنسيق JSON
  --dokan-debug                                تمكين مخرجات تصحيح Dokan
  --dokan-network <UNC>                        مسار UNC لوحدة تخزين الشبكة (مثال: \\host\myfs)
  --dokan-removable                            عرض وحدة التخزين كوسائط قابلة للإزالة
  --dokan-write-protect                        تركيب نظام الملفات للقراءة فقط
  --dokan-mount-manager                        تسجيل وحدة التخزين مع مدير التركيب Windows
                                               (يمكّن دعم سلة المحذوفات، إلخ)
  --dokan-current-session                      إظهار وحدة التخزين في الجلسة الحالية فقط
  --dokan-filelock-user-mode                   معالجة LockFile/UnlockFile في وضع المستخدم؛
                                               وإلا يديرها Dokan تلقائياً
  --dokan-enable-unmount-network-drive         السماح بإلغاء تركيب محرك الشبكة عبر المستكشف
  --dokan-dispatch-driver-logs                 إعادة توجيه سجلات برنامج التشغيل النواة إلى userland (بطيء)
  --dokan-allow-ipc-batching                   تمكين تجميع IPC لأنظمة الملفات البطيئة
                                               (مثال: التخزين البعيد)
  --public                                     انتحال هوية المستخدم المتصل عند فتح المقابض
                                               في CreateFile. يتطلب صلاحيات المسؤول
  --allocation-unit-size <بايت>                حجم وحدة التخصيص التي تبلغ عنها وحدة التخزين
  --sector-size <بايت>                         حجم القطاع الذي تبلغ عنه وحدة التخزين
  --volume-name <اسم>                          اسم وحدة التخزين المعروض في المستكشف (افتراضي: EncFS)
  --volume-serial <hex>                        الرقم التسلسلي لوحدة التخزين بالست عشري (افتراضي: من الأساس)
  --paranoia                                   تمكين تشفير AES-256 وإعادة تسمية IVs وتسلسل IV الخارجي
  --alt-stream                                 تمكين تدفقات البيانات البديلة NTFS
  --case-insensitive                           إجراء مطابقة أسماء الملفات بدون تمييز حالة الأحرف
  --nameio-stream                              استخدام تشفير أسماء الملفات nameio/stream عند إنشاء
                                               وحدة جديدة (الافتراضي: nameio/block)
  --cloud-conflict                             تمكين معالجة ملفات تعارض السحابة
                                               (Dropbox، Google Drive، OneDrive). معطل افتراضياً
  --reverse                                    الوضع العكسي: عرض rootDir النص العادي مشفراً
                                               في mountPoint

أمثلة:
  encfs.exe C:\Users M:                                    # تركيب C:\Users كمحرك M:\
  encfs.exe C:\Users C:\mount\dokan                        # تركيب في مجلد NTFS C:\mount\dokan
  encfs.exe C:\Users M: --dokan-network \\myfs\share       # كمحرك شبكة مع UNC \\myfs\share
  encfs.exe C:\Data M: --volume-name "محرك آمن"            # تركيب باسم وحدة تخزين مخصص
  encfs.exe C:\Data M: --use-credential                    # استخدام كلمة المرور المحفوظة من المدير
  encfs.exe C:\Data M: --nameio-stream                     # إنشاء وحدة جديدة باستخدام nameio/stream
  encfs.exe C:\Data M: --cloud-conflict                    # التركيب مع دعم تعارض السحابة
  encfs.exe C:\encrypted --scan-invalid                    # فحص أسماء الملفات غير الصالحة (إخراج JSON)

لإلغاء التركيب، اضغط Ctrl+C في هذه الوحدة الطرفية أو نفذ:
  encfs.exe -u <mountPoint>
```
